# 决策伙伴 - 产品规格文档

> 独立 H5 应用 | 基于论文核心发现：交互风格×场景匹配

---

## 1. 产品定位

**一句话**：帮你做决策——左脑分析利弊，右脑听见心声

**核心洞察**：同一个决策需求，用户有时需要逻辑分析（功利路径），有时需要情感共鸣（体验路径）。两种模式不是非此即彼，而是可以随时切换的双视角。

**用户价值**：
- 做重大决策时不孤独，有AI陪伴分析
- 看清利弊的同时，也听见内心声音
- 避免纯逻辑分析的"冷血感"和纯情感倾诉的"无效空转"

## 2. 核心交互

### 2.1 主界面布局

```
┌─────────────────────────────┐
│     [模式指示器 + 切换按钮]    │
│      🧠 左脑模式 / 💗 右脑模式   │
├─────────────────────────────┤
│                             │
│     [决策问题展示区]           │
│   "我该不该接受这个offer?"     │
│                             │
├─────────────────────────────┤
│                             │
│     [AI响应内容区]            │
│   （模式不同，内容风格不同）     │
│                             │
├─────────────────────────────┤
│  [输入框] 请描述你的决策困境... │
│                    [发送] ➤  │
└─────────────────────────────┘
```

### 2.2 模式切换方式

**方式一：滑动（主要）**
- 用户在屏幕主体区域**向左滑** → 切换到**左脑模式**
- 用户在屏幕主体区域**向右滑** → 切换到**右脑模式**
- 滑动手势需超过屏幕宽度的 30% 才触发切换
- 切换时有平滑的位移动画（300ms ease-out）

**方式二：点击模式指示器（辅助）**
- 点击顶部模式标签直接切换
- 底部模式切换按钮

**方式三：键盘快捷键（调试/PC端）**
- `←` 左箭头键 → 左脑模式
- `→` 右箭头键 → 右脑模式

### 2.3 模式视觉差异

| 维度 | 左脑模式 🧠 | 右脑模式 💗 |
|------|-----------|-----------|
| 主色调 | 冷蓝 `#4A90D9` | 暖粉 `#E8738A` |
| 背景渐变 | 蓝紫微光 | 粉橙微光 |
| 边框效果 | 冷静直线 | 柔软波浪 |
| 模式图标 | 🧠 | 💗 |
| 打字光标 | ▌ | ❀ |

## 3. 功能详细设计

### 3.1 问题输入

用户输入自己的决策困境，例如：
- "该不该辞职去留学？"
- "应该选择薪资高但加班多的，还是薪资低但稳定的？"
- "要不要和TA表白？"

**输入框 placeholder**：描述你的决策困境...

**快捷问题（可选）**：
提供 3-4 个常见决策示例按钮，新用户可以快速体验

### 3.2 左脑模式 🧠 —— 功利/分析路径

**风格**：冷静、理性、结构化、像一位资深商业顾问

**回复框架**：
```
【利弊分析】
✓ 优势：
  • [列出优势1]
  • [列出优势2]

✗ 劣势：
  • [列出劣势1]
  • [列出劣势2]

【关键因素打分】
  因素A：██████░░ 6/10
  因素B：████████░ 8/10
  ...

【决策建议】
基于以上分析，...
```

**语气特点**：
- 使用"基于以上分析"、"从理性角度看"
- 避免情绪化词汇
- 主动提问："你更看重【稳定】还是【发展】？"

### 3.3 右脑模式 💗 —— 体验/共情路径

**风格**：温暖、陪伴、像一位懂你的老朋友

**回复框架**：
```
【我听见你了】
[先复述用户的困境，表达理解]
听起来你现在很纠结...

【你真正在担心的是什么？】
[深入一层，挖掘情绪]
也许你不只是在担心【offer本身】，而是在担心...

【跟随你的感觉】
[引导用户听见内心答案]
如果闭上眼睛想象做出选择之后的那一天...
你感觉到的第一个情绪是什么？

【陪伴结语】
无论你选哪个，记得你值得被好好对待。
```

**语气特点**：
- 使用"我听见你"、"我理解你的纠结"
- 允许沉默和不确定
- 不给标准答案，而是帮用户发现自己内心的声音

### 3.4 模式联动

两种模式的回复是互补关系，不是对立的：

```
用户问："该不该接受这个offer?"

→ 左脑回复：你应该从职业发展、薪资、环境等维度打分...
→ 右脑回复：我听见你的纠结了，这个选择背后是不是还有...

用户可以随时切换模式，获得另一视角
```

### 3.5 对话历史

- 保存本次会话的问答历史（localStorage）
- 刷新页面后清空
- 新问题开启新对话

## 4. 技术方案

### 4.1 技术栈

- **单文件 H5**：`index.html`（HTML + CSS + JS，无框架）
- **AI 接口**：百炼 API（与论文实验共用）
- **无需后端**：纯前端，直接调用百炼 API

### 4.2 API 调用设计

**端点**：`https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions`

**请求格式**：
```json
{
  "model": "qwen-plus",
  "messages": [
    {"role": "system", "content": "[模式系统提示词]"},
    {"role": "user", "content": "[用户问题]"}
  ],
  "temperature": 0.7,
  "max_tokens": 800
}
```

**系统提示词（左脑）**：
```
你是一位专业、冷静的商业决策顾问。用户会描述一个决策困境，你需要：
1. 结构化地分析利弊（优势/劣势）
2. 识别关键因素并量化打分
3. 给出客观建议，但不强加决定
4. 最后问一个关键问题，引导用户深入思考
语气：理性、专业、简洁。不使用emoji。
```

**系统提示词（右脑）**：
```
你是一位温暖、陪伴的朋友。用户会描述一个决策困境，你需要：
1. 先复述并确认你听懂了用户的困境
2. 深入挖掘情绪——这个选择背后，用户真正担心的是什么？
3. 帮助用户听见自己内心的声音
4. 不给"正确答案"，而是陪伴用户发现自己的答案
语气：温暖、柔和、有同理心。允许沉默和不完美。可以用emoji点缀。
```

### 4.3 滑动交互实现

- 使用 Touch Events（touchstart/touchmove/touchend）
- 计算滑动距离，超过阈值触发模式切换
- 切换时添加 CSS transition 动画
- 防止页面整体滚动（scroll 事件 preventDefault）

### 4.4 响应式设计

- 移动端优先（375px 基准）
- 最大宽度 480px（居中显示）
- 触摸目标最小 44px × 44px

## 5. 交互细节

### 5.1 加载状态

- 用户发送问题后，输入框显示"思考中..."
- 左脑模式：显示蓝色脉冲动画
- 右脑模式：显示粉色渐变动画

### 5.2 打字机效果

- AI 回复采用打字机效果逐字显示
- 左脑模式：标准打字速度
- 右脑模式：稍慢，更温暖

### 5.3 空状态

用户还没输入问题时，展示引导界面：

```
┌─────────────────────────────┐
│      🧠 💗 决策伙伴          │
│                             │
│  重大决策前，听听左右两个声音    │
│                             │
│  ┌───────────────────────┐  │
│  │ "我该辞职去留学吗？"   │  │
│  └───────────────────────┘  │
│                             │
│  [示例问题] [示例问题] [示例问题]│
│                             │
│  👆 在这里输入你的决策困境      │
└─────────────────────────────┘
```

### 5.4 错误处理

- API 调用失败：显示"网络开小差了，请稍后重试～"（右脑风格）
- 空输入发送：输入框轻微抖动，不触发请求
- 超时（30s）：显示"思考时间有点长，换个问题试试？"
