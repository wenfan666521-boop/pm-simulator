# 决策双生 - 产品规格文档 v2

> 独立 H5 应用 | 基于论文核心发现：交互风格×场景匹配

---

## 1. 产品定位

**一句话**：帮你做决策——左脑分析利弊，右脑听见心声

**核心洞察**：同一个决策需求，用户有时需要逻辑分析（功利路径），有时需要情感共鸣（体验路径）。两种模式不是非此即彼，而是可以随时切换的双视角。

---

## 2. 左脑模式（🧠 理性分析路径）

### 2.1 流程总览

```
用户输入问题
    ↓
AI 分析（1-2轮）
生成 3-6 个选项 + 初始权重 + 简要理由
    ↓
权重调节界面
用户拖动滑块调整各选项权重
实时饼图可视化（归一化）
    ↓
点击"开始决策" → 加权转盘旋转
    ↓
结果卡片 + AI 分析理由
```

### 2.2 权重调节界面

**UI 布局：**
```
┌──────────────────────────────────┐
│ 【分析完成】你的纠结可以归结为：    │
│                                  │
│ 选项列表（每项一行）：             │
│ 🅰 选项A ▓▓▓▓▓▓▓░░░ 35% [滑块]  │
│ 🅱 选项B ▓▓▓▓▓░░░░░░ 25% [滑块]  │
│ 🅲 选项C ▓▓▓▓░░░░░░░ 20% [滑块]  │
│ 🅳 选项D ▓▓░░░░░░░░░ 20% [滑块]  │
│                                  │
│ [+ 添加选项]  (最多6个)           │
│                                  │
│      ╭─────────╮                 │
│     ╱  实时饼图   ╲               │
│    │   (实时更新)  │              │
│     ╲           ╱               │
│      ╰─────────╯                 │
│                                  │
│   [ 🎯 开始决策 ]                │
└──────────────────────────────────┘
```

**交互规则：**
- 每个选项旁边有一个滑块，拖动调整权重比例
- 调整后自动归一化（总和=100%），其他选项等比压缩
- 饼图实时更新，扇区按权重比例
- 点击选项名可编辑
- 每个选项旁有删除按钮
- 最多 6 个选项
- 初始权重由 AI 分析逻辑生成

**归一化逻辑：**
用户拖动任意滑块 → 计算调整差值 → 将差值等比分配/压缩到其他选项 → 保证总和=100%

### 2.3 加权转盘

**视觉设计：**
- Canvas 绘制饼图，每个扇区对应一个选项
- 扇区颜色使用固定调色板（6种颜色循环）
- 扇区大小按权重比例
- 悬停扇区高亮，显示选项名 + 权重%
- 中心显示"开始决策"按钮

**旋转动画：**
- 点击后转盘旋转 3-5 圈
- 使用 ease-out 缓动，模拟物理惯性减速
- 旋转结束 → 指针指向的扇区为结果
- 结果扇区高亮闪烁

### 2.4 结果卡片

**展示内容：**
```
═══════════════════════
    🎯 决策结果：A
    ─────────────
    占比 35%

    AI 分析理由：
    （AI 根据原始分析写的1-2句话）
═══════════════════════

[ 重新分析 ] [ 💗 右脑看看 ] [ 再转一次 ]
```

## 4. 通用界面

### 4.1 模式切换
- 主体区域左滑 → 左脑模式
- 主体区域右滑 → 右脑模式
- 也支持点击模式标签切换

### 4.2 对话历史
- 保存本次会话的问答历史（sessionStorage）
- 刷新页面后清空

---

## 5. 技术方案

- 单文件 H5（HTML + CSS + Canvas JS）
- AI 接口：DeepSeek API
- 画布绑制转盘（Canvas 2D）
- 响应式：最大宽度 480px
